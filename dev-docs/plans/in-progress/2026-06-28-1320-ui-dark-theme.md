# UI Overhaul — Dark Theme, Chrome-less Toolbar, Project Frames
> 2026-06-28

## Context

The app works (canvas, panels, terminals, sidebar, editor stub) but looks like a default
light AppKit app. The user wants a cohesive dark, terminal-like aesthetic modeled on
Xcode/Mail: a chrome-less window with a tall unified toolbar, a dark vibrancy sidebar, custom
blue-gray colors, consistent sidebar selection, Xcode-width sidebar, and a per-project boundary
frame on the canvas.

Decisions from the user:
- Sidebar uses **native macOS vibrancy** under a dark appearance (authentic Xcode/Mail gradient
  transition; color ≈ #212029, not pixel-exact).
- Fat top toolbar contains: **sidebar toggle, current project name, zoom controls, add (+) menu**.

Color palette (sRGB):
- Canvas `#4D4A5F` (77,74,95) · Sidebar tint `#212029` (33,32,41) · Project frame `#716C90`
  (113,108,144).

## Implementation

### 1. Palette — new `EndlessTerminal/Support/Palette.swift`
`enum Palette` with `NSColor(srgbRed:…)` constants: `canvas`, `sidebarTint`, `projectFrame`,
plus derived panel chrome (`panelBody` ≈ #29283A dark, `panelTitleBar` ≈ #38364A,
`panelBorder` = a muted #716C90), and `gridDot` (white α≈0.06). Single source of truth; replace
the hard-coded `NSColor(calibratedWhite:…)` values scattered in the views.

### 2. Chrome-less window + fat unified toolbar — `App/MainWindowController.swift`
- Add `.fullSizeContentView`; `titlebarAppearsTransparent = true`; `titleVisibility = .hidden`;
  `window.appearance = NSAppearance(named: .darkAqua)` (this alone darkens the toolbar, sidebar
  vibrancy, outline view, scrollers, and controls).
- Attach an `NSToolbar` (`toolbarStyle = .unified` for the tall/"fat" look) with
  `MainWindowController` as delegate. Items:
  - `.toggleSidebar` (system) → collapses sidebar.
  - `NSTrackingSeparatorToolbarItem(identifier:splitView:dividerIndex: 0)` so the toolbar divider
    tracks the sidebar/canvas split (the Xcode seam).
  - **Project name**: custom item hosting an `NSTextField` label; updated via
    `model.onCurrentProjectChange` and on load.
  - `.flexibleSpace`.
  - **Zoom**: custom item with an `NSSegmentedControl` (−, 1:1, +) → `zoomOut/zoomReset/zoomIn`.
  - **Add (+)**: custom item, `NSButton` with `plus` symbol that pops a menu (New Terminal /
    Document / Project / Open File…).
- Toolbar actions route through the responder chain to `MainSplitViewController` (existing
  `zoomIn/out/reset`, `newTerminal/newDocument/newProject`; add `openDocument`). Custom item view
  heights ~32 to fatten; tunable.
- `MainWindowController` now creates the `AppModel` once and shares it with the split VC and the
  toolbar (currently the model is created inline in the controller — hoist it to a stored prop).

### 3. Canvas + panel theming
- `Canvas/CanvasView.swift`: `backgroundColor = Palette.canvas`, `dotColor = Palette.gridDot`.
- `Canvas/CanvasScrollView.swift`: `backgroundColor = Palette.canvas` (overscroll matches).
- `Windows/WindowView.swift`: body/title/border use `Palette.panelBody/panelTitleBar/panelBorder`;
  keep the existing rounded-rect + shadow drawing.
- `Content/DocumentPanel.swift`: tweak `EditorTheme.endlessDark.background` to a dark shade that
  reads against the lighter canvas.

### 4. Sidebar — `Sidebar/SidebarViewController.swift`
- Keep `NSSplitViewItem(sidebarWithViewController:)` (vibrancy) + `outlineView.backgroundColor =
  .clear`, `scrollView.drawsBackground = false`. Dark appearance (from §2) gives the dark
  gradient transition against the lighter canvas automatically.
- **Consistent selection**: add a `SidebarRowView: NSTableRowView` returned from
  `outlineView(_:rowViewForItem:)` that draws an identical rounded-rect selection highlight for
  *every* row regardless of level (project vs terminal vs document), removing today's
  project-vs-item mismatch. Use the accent/`Palette` highlight uniformly; keep icon + label cell.
- **Width to match Xcode**: `sidebarItem.minimumThickness = 220`, `maximumThickness = 400`; after
  adding split items call `splitView.setPosition(260, ofDividerAt: 0)` and set
  `splitView.autosaveName` so it persists (~260 ≈ Xcode navigator).

### 5. Per-project boundary frame — `Canvas/CanvasView.swift`
- In `draw(_:)` (behind the window subviews) draw a **2px `Palette.projectFrame` rectangle** =
  the union of all window frames inset by **−80pt** on every side (80px padding around content).
  When the project has no windows, draw a default-sized box centered in `visibleRect`.
- Recompute live: add `WindowView.onGeometryChange` invoked from `mouseDragged` (move/resize) and
  `setFrameSize`; `CanvasView.addWindow` sets it to `setNeedsDisplay(...)`. Also redraw on
  add/remove. (User-resizable frames are a later step; this delivers the auto-padding box now.)

### 6. Finish pending milestone-4 wiring (open/save)
`DocumentPanel.swift` exists but isn't wired. Complete per
`dev-docs/plans/planning/2026-06-28-1315-milestone-4-editor.md`: `WorkItem.document`,
`AppModel.activeDocumentItem`, `addItem(kind:url:)` document case + `onFocus` wrap, and
**Open… ⌘O / Save ⌘S / New Document ⌘N** (now surfaced via the toolbar + menu add the menu).

## Verification

1. `xcodegen generate` then `xcodebuild … build`; launch.
2. **Chrome/toolbar**: no white title bar; traffic lights sit over the dark UI; a tall unified
   toolbar shows sidebar-toggle, project name, zoom, and +; toggle hides the sidebar; the toolbar
   divider tracks the split.
3. **Colors**: canvas reads ≈ #4D4A5F, sidebar dark vibrancy with the gradient transition to the
   canvas; panels and editor are dark and legible.
4. **Sidebar**: select a project, then a terminal, then a document — selection highlight is
   identical for all; sidebar width ≈ 260 and persists across relaunch.
5. **Project frame**: a 2px #716C90 box surrounds the project's windows with 80px padding and
   updates live as windows are dragged/resized; empty project shows a default box.
6. **Editor**: + → Open File… opens a file with highlighting; ⌘S saves; New Document + ⌘S prompts
   for a location.
